私達のロボトリオはゴルフをしたいのですが、 彼らはストロークが正確すぎて何の問題もなくホールを回れてしまいます。 なので、彼らはルールを変更してよりチャレンジングになるようにしました。 彼らはどのような順番でもいいので各ホールでボールを通過させる必要があります。

10x10の大きさのユニットを持つ四角の場所のに5つのホールがあります。 あなたはホールの座標が与えられます。 勝つためにあなたはすべてのホールを回る最も短い道を探し、あなたの関数はその道の長さを返す必要があります。 あなたは(0, 0)の場所からスタートし、どのホールで終わっても良いです。 結果は±0.01の精度でチェックされます。

このミッションでの主なゴールはあなたのコードを可能な限り短くすることです。 コードが短いほど、よりポイントを獲得します。 あなたのスコアは動的でコードの長さに直接関連しています。 参考までに、スコアは使われた文字の数に基いています。 350文字が最大許容数で、それは0ポイントの獲得になります。 350から1文字づつ減らす毎に1ポイント獲得します。例えば300文字の長さのコードは50ポイント獲得します。

入力 ホールの座標の集合。それぞれの座標は整数のタプルとして表される。

出力 最も短い道の長さをfloatとして返す。

例 

golf({(2, 2), (2, 8), (8, 8), (8, 2), (5, 5)}) == 23.31
golf({(2, 2), (4, 4), (6, 6), (8, 8), (9, 9)}) == 12.73

どうやって使われるか このタスクはエレガントやパフォーマンスコードを書くことではありません。 これはPythonにおけるコードを短くするハックとトリックについてです。 あなたはこれを製品に使う必要はありませんが、Pythonの深い理解に役立つかもしれません。

事前条件 len(holes) == 5
all(0 < x < 10 and 0 < y < 10 for x, y in holes)
